| Var | State before | State after |
|---|---|---|
| `ANTHROPIC_API_KEY` | **missing entirely** from prod | added (production + development) |
| `OPENROUTER_API_KEY` | empty string | re-added with `--value --force` |
| `HUNTER_API_KEY` | literal `\n` suffix | re-added clean |
| `NEXT_PUBLIC_SUPABASE_URL` | literal `\n` suffix → caused prod /send 404 because RLS reads silently failed | re-added clean |

**Detection rule for next time:** when prod fails silently but local works, run `vercel env pull /tmp/check --environment=production --yes` and grep `\\n` on every value. Note that `vercel env pull` shows sensitive vars as `""` by default — check `vercel env ls` for "Encrypted" presence instead. Use `vercel env add NAME env --value "$VAR" --force --yes` (the `printf | vercel env add` stdin pattern is unreliable).

## Hard rules — DO NOT BREAK

From `CLAUDE.md` in the repo root, verbatim from Tristan:

> "There is no permission to automatically send things until they have been approved. That has to be a very fast rule that cannot be broken."

Three layers of defence in depth:
